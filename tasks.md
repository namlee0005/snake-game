`event.preventDefault()` only for arrow keys (prevent page scroll). Not for Space (allow default browser behavior to be overridden at call site).

```js
/**
 * Binds click/touch events to four D-pad button elements.
```

## Phase 5 — Renderer

### T5.1 — Inline in `game.js` (not a separate module — only called from game loop)

```js
/**
 * Full canvas redraw each tick (20×20 grid = 400 cells — negligible cost).
 * @param {CanvasRenderingContext2D} ctx
 * @param {{ cols:number, rows:number, cellSize:number }} config
 * @param {{ snake: object, food: {x:number,y:number} }} state
 */
function render(ctx, config, state)
```

Draw order: `clearRect` → food → snake body segments → snake head (distinct color).

**No OffscreenCanvas.** Regular Canvas is sufficient for 400px at 60fps. OffscreenCanvas reduces Safari compatibility and adds zero measurable benefit here.

Color tokens (CSS custom properties on `:root`, read via `getComputedStyle` once at init):
```
--c-bg:         #1a1a2e
--c-snake-head: #e94560
--c-snake-body: #0f3460
--c-food:       #f5a623
```

## Phase 6 — Game Loop

### T6.1 — `src/game.js`

**Config (module-level constant):**
```js
const CONFIG = Object.freeze({
  cols: 20,
  rows: 20,
  cellSize: 20,        // px → canvas 400×400
  baseSpeed: 150,      // ms per tick
  minSpeed: 60,        // ms floor (~16fps equivalent)
  speedStep: 5,        // ms reduction per food eaten
})
```

**Speed formula:**
```js
// On each food eaten:
state.speed = Math.max(CONFIG.minSpeed, state.speed - CONFIG.speedStep)
```

**Game loop — `requestAnimationFrame` with timestamp accumulator:**
```js
let last = 0, elapsed = 0

function loop(now) {
  elapsed += now - last
  last = now
  while (elapsed >= state.speed) {
    tick()
    elapsed -= state.speed
  }
  render(ctx, CONFIG, state)
  requestAnimationFrame(loop)
}
```

**Critical:** Subtract `state.speed` (not `elapsed`) in the while loop — prevents spiral of death on tab resume where elapsed could be thousands of ms.

**`tick()` sequence:**
1. Apply `snake.nextDir` → `snake.dir`
2. `moveSnake(snake)`
3. `collidesWithWall` or `collidesWithSelf` → `state.phase = 'over'`; `saveHighScore(state.score)`; `ui.showOverlay('over', state.score)`; return
4. Head position === food position → `growSnake`, `spawnFood`, `state.score++`, recalculate speed, `ui.updateScore`
5. If `saveHighScore(state.score)` returns `true` → `ui.updateHighScore(state.score)`

**Phase transitions (Space key / D-pad tap on overlay):**
```
'idle'    → 'running'  (initialize fresh state, start loop)
'running' → 'paused'
'paused'  → 'running'
'over'    → 'running'  (reinitialize state, reuse loop)
```

**Entry point:**
```js
/** Called once on DOMContentLoaded. */
export function initGame(canvas)
```

Validates `canvas.getContext('2d')` — shows an error message and exits if unavailable (unlikely but non-zero risk on some older WebViews).

### T7.2 — `src/styles.css`

Layout requirements:
- CSS custom properties on `:root` for all colors
- `#canvas` parent: `max-width: min(400px, 100vw); aspect-ratio: 1` — canvas never CSS-scales, container shrinks
- `#overlay`: `position: absolute; inset: 0` over canvas; `opacity` transition 200ms; hidden by default
- `#dpad`: CSS grid layout `3×3`; center button empty; visible only on `@media (hover: none)` (touch devices)
- `touch-action: manipulation` on all `#dpad button` elements
- `@media (prefers-reduced-motion: reduce)` — set `transition: none` globally

- Smooth movement interpolation (defeats classic feel)
